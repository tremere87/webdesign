2.1 Review the ongoing use of IT tools and change approach as needed– in making your own website, are you happy with the tools you used? E.g. Visual Studio Code, Codepen, Trello

A major issue that was discovered with the Visual Studio Code was that there was translating issues in my coding. Even though my experience with website design was very limited before starting the course, i have nevertheless wanted to apply coding and content more complex than my ability to code. Also when you’re reaching 300 lines of code, it becomes very problematic when one “.” or "p” element, out of place, can suddenly stop everything from working.

Trello is a useful tool for monitoring your progress throughout a project as intended, though can equally prove a very daunting tool, if you have a very short deadline for a project, with many small tasks assigned to the 1st “needs doing” bar, set up.

Slack whilst not heavily used, since this is a solo project, has been useful to get the occasional bit of useful code sent over to me by the instructor without having it dictated to me, etc. Moodle however has been instrumental, as a lot of the tasks we have been trying to apply to our websites are brand new techniques for us, so we’ve been learning as we go.

The website works, which is as good as can be expected at this stage, There are a few issues I have with it and features I would’ve liked to put in, but alas with the time constraints and lack of experience, I would fail to meet the deadline, if I tried to implement them in the coding. Originally I had intednded to use a background that would've made the text look hard to read in anything other than bright red, which i feel would've looked tacky and garish. The idea would've been to use a light grey border for the body of text. The text-box would've been semi-transparent and allowed black text be written in those boxes, however despite several different attempts, re-edits and 1 "restart from previous save-filer", the text boxes couldn't be reliably coded over the background image.


As the example above shows, i was successful in getting the text box to show as grey, but the areas it coded and the margin/parameters as well as the sections of text it was displaying for could not be assured. Suffice to say too much time was being wasted trying to fix this problem than should've been, which jeapordised the time i had avaulable to complete the content for the site

3.1 Review the benefits and drawbacks of IT tools and systems used in terms of productivity and efficiency – could you have been more efficient when making your website? How? Could you have done it in teams?

The answer to a question like this is nearly always going to be yes; in this case the obvious factors that would have improved efficiency in both the website’s design as well as the time in which it got coded, is experience. With more experience in the subject matter I would have been faster writing up the answer and wouldn’t have had so many code-issues that needed troubleshooting.

1 major drawback for me personally when designing this website is a particular dislike for the Apple/IOS interface and nearly 25years of using a windows (...well DOS originally, but we won’t go there) PC, meaning it has been an initial learning challenge to familiarise myself with the device, before being able to do any kind of coding on it.

Working in teams would have presented new issues to overcome, as it introduces difference of styles in both aesthetic and writing of code. As VSCode has shown to be very specific how it translates commands, functions etc, based on the order it is prioritised, it could cause major errors to occur if different coders were to try and compile their separate workings into a final product. To ensure success, a constant would have to be set prior to the start of the project

3.2 Describe ways to improve productivity and efficiency

I personally would have been more efficient working in a different environment to the one i have done for the production of this project. Due to daily medication I must take, which raises my metabolism/body temperature, i am not able to function in particularly warm work-conditions, such as the office block that we had to.

From this project I have noted that there have been others who equally struggled in the heat during the summer conditions. From my personal experiences, I would surmise that the productivity and focus of the whole coding team would drastically improve, when they are comfortable in their workspace.

3.3 Develop solutions to improve own productivity using IT in digital projects – what would you do differently next time? Team work? Use of Slack?

For the aforesaid reasons i feel that if teamwork was goihg to be included in the project then the best way to implement it, would have been to compile the data to be added to the site as a group, share information and ideas over slack and write content up on something like google.docs, which is a generic writing-platform accessible from any device. When all the data is shared and compiled, i'd have it sent to one team member in order to code it, (as guaranteeing a standardised coding style across multiple group member's contribution is hard to ensure).

3.4 Describe how you would go about testing digital solutions?

Since googling "testing digital solutions" brings up a list of sites and companies which are focused specifically to do the job. For an amateur site, meant for a course project, like this; i would most likely get a test group from my coursemates and then do a more generalised audience to test it out. I find the more inexperienced or "obtuse" a test-user is, the more likely they are to find a flaw in what was considered a "fool-proof project"
